The first step we tackled about was to figure out which actions the user would be able to perform with our program.
First of all, in order to carry out this, we relied on the Use case diagram made with UML (the Unified Modeling Language) in order to specify the functionalities we wanted to offer with our software.

Use case name

Actors
  • User
  • European Union API

Use case #6

Use case name

Search Service

Actors
  • User
  • European Union API

Description

The user searches a service selecting countries, providers, types and status.

Preconditions

Use case #7 must be executed with success

Flow
  1. The user clicks on “Search service” button triggering the use case
  2. The system shows the countries list
  3. The user selects one or more countries and clicks on “Next” button
  4. The system shows the types of services based on the previous selections
  5. The user selects one or more providers and clicks on “Next” button
  6. The system shows the types of services based on the previous selections
  7. The user selects one or more types and clicks on “Next” button
  8. The system shows the status of services based on the previous selections
  9. The user selects one or more status and clicks on “Next” button
  10. The system shows all services matching the previous selections

Postconditions

The user is viewing a services list based on its selections.

Domain model

After dealing with the creation of use cases, the next step was to sketch out a domain model.
A Domain model is a structured visual representation of interconnected concepts or real-world objects that incorporates vocabulary, key concepts, behavior, and relationships of all of its entities.

While doing the domain model, we analyzed what entities were in our sphere of interest and how we could use them.
We found concrete entities (such as nations, services, providers, types, and status) but we decided to focus on abstract entities such as search and selection.
This choice was made since we wanted to create a software that would allow Trusted Services navigation and search, therefore we concluded that parameter selection (e.g. nation, service etc) was particularly important in our plans.

Features

The aim of our program is to allow users to view the list of European Union countries and their related providers and services.
This functionality was achieved through the use of the Rest API provided by the European Union to obtain the data. These data were then used to create the interface to interact with the user.
The program in particular was created through the use of JavaFX, a library that allows the creation of interfaces in the Java language.
